# Инструкция по установке и запуску AstroMatch

## Требования

- **macOS** 12.0 или новее
- **Xcode** 14.0 или новее
- **iOS Simulator** или физическое устройство с iOS 15.0+

## Установка

### 1. Клонирование репозитория

```bash
git clone <URL_РЕПОЗИТОРИЯ>
cd AstroMatch
```

### 2. Открытие проекта в Xcode

1. Запустите Xcode
2. Выберите "Open a project or file"
3. Найдите папку `AstroMatch` и выберите файл `AstroMatch.xcodeproj`
4. Нажмите "Open"

### 3. Настройка проекта

1. В Xcode выберите проект `AstroMatch` в навигаторе
2. Выберите target `AstroMatch`
3. В разделе "Signing & Capabilities":
   - Выберите вашу команду разработчиков
   - Убедитесь, что Bundle Identifier уникален
4. В разделе "Deployment Info":
   - Убедитесь, что iOS Deployment Target установлен в 15.0

### 4. Запуск приложения

1. Выберите симулятор iPhone в верхней части Xcode
2. Нажмите кнопку "Run" (▶️) или используйте сочетание клавиш `Cmd + R`
3. Дождитесь компиляции и запуска приложения

## Возможные проблемы и решения

### Ошибка компиляции

1. Убедитесь, что Xcode обновлен до последней версии
2. Очистите проект: `Product` → `Clean Build Folder`
3. Перезапустите Xcode

### Ошибки подписи

1. Проверьте настройки в разделе "Signing & Capabilities"
2. Убедитесь, что у вас есть действующий сертификат разработчика
3. Попробуйте использовать автоматическую подпись

### Проблемы с симулятором

1. Сбросьте симулятор: `Device` → `Erase All Content and Settings`
2. Перезапустите симулятор
3. Убедитесь, что выбран правильный симулятор

1. **Аутентификация**

   - Вход в существующий аккаунт
   - Выход из аккаунта

2. **Поиск партнеров**
   - Просмотр списка совместимых партнеров
   - Применение фильтров
   - Поиск по имени

3. **Анализ совместимости**
   - Выбор пользователя для анализа
   - Просмотр результатов совместимости
   - Чтение рекомендаций

4. **Профиль пользователя**
   - Просмотр профиля
   - Редактирование информации
   - Настройки приложения

### Архитектура

Приложение использует архитектуру MVVM (Model-View-ViewModel) с SwiftUI:

- **Models**: Структуры данных
- **Views**: Пользовательский интерфейс
- **ViewModels**: Бизнес-логика и состояние
- **Services**: Внешние сервисы и API
